About the country

Addresses in Brazil typically follow the following structure:
{name} Destinatário The name is often split into {first-name} (Nome) and {family-name} (Sobrenome).
{street}, {building} Endereço, Número The street name and house-number typically appear as separate fields.
{address-overflow} {landmark} Complemento e Ponto de referência Address forms often contain a field for extra information that is optional and can be used for apartments (apto), block numbers (bloco), room numbers (sala), floor numbers (andar). Address forms also often contain a landmark field ("Ponto de referência"). Some forms combine both into a single field.
{locality2} Bairro Most address forms contain a field for the neighborhood.
{locality1} - {admin-area1} Cidade - Estado Most address forms contain a field for the neighborhood and state.
{postal-code} CEP Most address forms contain a postal code field.
{country} País/Região Some address forms contain a country field.

Overview of concepts

name - Full name | Destinatário
given-name - First name | Nome
family-name - Last name | Sobrenome
 
address - Address of a physical location - Artificial concept, not to be used in HTML
street-address - Street address (street and location within the street) - this is very rare on Brazilian websites.
address-line1 - 1st line of street-address
address-line2 - 2nd line of street-address
address-line3 - 3rd line of street-address
address-line4 - 4th line of street-address
street-address-alternative-1 - Artificial concept, not to be used in HTML; this is a structured representation of a street address
street-location - Name of a street and identifier for the building
street - Name of a street | Endereço
building - House number | Número da residência
address-overflow-and-landmark
address-overflow - An overflow field for information that is not captured differently in a form | Complemento
in-building-location - Location of the adresse in a building
floor - Floor number in the building
unit - Apartment | Apto. (por exemplo "ap. 5")
unit-type - Type of a unit (e.g. "Apt", "Room", "Store")
unit-name - Identifier of a unit (e.g. "5")
landmark - Landmark | Ponto de referência
locality1 - City | Cidade
locality2 - Neighborhood | Bairro
admin-area1 - State | Estado
postal-code - Postal code | CEP
country - 2-letter country code (e.g. "BR")
country-name - Name of a country (e.g. "Brazil") | País
 
organization - Name of an organization/company | Empresa
 
credit-card - Artificial concept, not to be used in HTML
cc-name - Full name of credit card holder
cc-given-name - Given name of credit card holder
cc-additional-name - Additional name of credit card holder
cc-family-name - Family name of credit card holder
cc-number - Credit card number
cc-exp-MMYY - Credit card expiration date in format MM/YY
cc-exp-MM - Credit card expiration month (2 digits)
cc-exp-YY - Credit card expiration date in format YY (2 digits)
cc-exp-MMYYYY - Credit card expiration date in format MM/YYYY
cc-exp-MM - Credit card expiration month (2 digits)
cc-exp-YYYY - Credit card expiration date in format YYYY (4 digits)
cc-csc - CSC/CVC/CVV number
cc-type - Credit card type
 
tel - Full telephone number
tel-country-code - Country code component of the telephone number
tel-national - Telephone number without the county code component, with a country-internal prefix applied if applicable
tel-area-code - Area code component of the telephone number, with a country-internal prefix applied if applicable
tel-local - Telephone number without the country code and area code components
tel-extension - Telephone number internal extension code
 
email - Email address
